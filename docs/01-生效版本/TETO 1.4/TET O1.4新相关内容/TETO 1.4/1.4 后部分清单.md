# 1.4 后部分清单

我们之前是在用“贴标签”的思维做 1.4（给记录贴上计划、现在、心情、能量的标签），但你现在的思路是 **“语义解析与关联图谱”** 的思维，这非常契合 1.4“连续人生现实结构深化”的主题。

你实际上是在定义一套​ **“人生记录的语法（Grammar of Life）”** 。

顺着你的 4 点，我们重新梳理现在的核心任务：

### 1. 从“推断发生/计划”升级为 👉 “时间锚点与记录关联” (对应你的第1点)

你的照片比喻很准。记录里提到的不是空泛的“计划”，而是自带**时间锚点（Time Anchor）** 的：

- “明天吃猪脚饭” → 提取出时间实体：`Date: Tomorrow`​，行为：`eat pork leg rice`。（向未来抛出锚点）
- “上周考试90分” → 提取出时间实体：`Date: Last week`​，并触发​**记录关联（Record Linking）** ，去寻找“上周的考试记录”并建立连接。（向过去抛出锚点）
- ​**结论**：我们需要让 AI 在处理输入时，自动识别“时间指向”和“关联事件”，而不是简单地标个“计划”属性。

### 2. 从“提取心情/状态”升级为 👉 “记录的语法解构” (对应你的第2点)

你提到了主谓宾定状补、地点、关系人。这意味着一条记录不再是 `text + 标签`，它是一个结构化的断言：

- ​**主语 (Subject)** ：我 / 父母 / 朋友（关系人）
- ​**谓语 (Action)** ：吃 / 考试 / 跑步
- ​**宾语 (Object)** ：猪脚饭 / 驾照科目一
- ​**状语 (Context)** ：在公司（地点） / 上周（时间） / 很高兴地（心情/状态） / 疲惫地（能量）
- ​**结论**​：心情、能量只是修饰谓语或主语的“状语/定语”，而不是独立乱飞的标签。我们需要定义一个清晰的​**记录结构模型（Schema）** 。

### 3. 需要算法吗？关于复合句拆分 (对应你的第3点)

**绝对需要一套逻辑，但不需要我们从零手写传统的语法分析器（AST）。**   
因为我们有 LLM（大语言模型），它天然就是最强的语义分析器。我们的“算法”应该是：  
**制定一套极其严密的 prompt 规则和输出 JSON 结构（Schema）。**   
遇到复合句（比如：“昨天和小明去爬坡，很累，但想到明天要吃猪脚饭又很开心”）：

1. 算法层面先用 LLM 将长句根据动作主体和时间锚点​**切分成独立的短句模型**（爬坡事件 / 期待猪脚饭事件）。
2. 再用第 2 点的“语法解构”去填充每个子事件的要素。
3. 形成多条关联记录同时入库。

### 4. 搁置目标与洞察 (对应你的第4点)

同意。如果“单张照片”的要素（时间锚点、语法解构）没解析清楚，相册的目标（事项）和整个图库的洞察（总结）就全是无源之水。先把底层记录的结构解析打磨透。
