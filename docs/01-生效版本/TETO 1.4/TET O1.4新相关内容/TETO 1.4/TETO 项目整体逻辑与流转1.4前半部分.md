# TETO 项目整体逻辑与流转1.4前半部分

你现在讲一下整个项目的逻辑，怎么使用，怎么流转

### 一、项目定位

​**TETO \= 个人效率系统**​。核心理念是 ​ **"记录驱动"** ——一切从记录开始，事项和洞察都是记录的聚合结果，而不是反过来。

### 二、三页架构

```
plaintext记录页 ──→ 事项页 ──→ 洞察页
 (输入)     (组织)     (反馈)
```

#### 1. 记录页 ﻿/records﻿ — 第一入口

​**用法**：随手记下发生的事。比如"背了30个单词"、"跑步5公里"、"开会讨论需求"。

​**两种模式**：

- ​**单日模式**：一次看一天，左右切日期，聚焦当天
- ​**多日模式**：横向并排看多天（类似看板），可以"更早"/"更晚"按钮渐进加载更多天

​**记录的结构化字段**（P1新增）：

|字段|含义|示例|
| ----------------------------| ---------------------------| ----------------|
|﻿﻿content﻿|记录内容（必填）|"单词复习30个"|
|﻿﻿type﻿|类型：发生/计划/想法/总结|发生|
|﻿﻿metric\_name﻿|统计对象|单词|
|﻿﻿metric\_value﻿|统计数值|30|
|﻿﻿metric\_unit﻿|计量单位|个|
|﻿﻿duration\_minutes﻿|时长|25|
|﻿﻿cost﻿|花费金额|—|
|﻿﻿item\_id﻿|关联到哪个事项|英语单词复习|

​**交互流程**：

1. 顶部 QuickInput 输入内容 → 点击发送
2. 可展开填写统计数值（对象+数值+单位）、时长等
3. 点击已有记录 → 右侧 Drawer 编辑详情
4. 可选关联到某个事项（﻿item\_id﻿）

#### 2. 事项页 ﻿/items﻿ — 组织容器

​**用法**：把零散的记录归类到事项下，形成持续追踪的主题。比如"英语单词复习"、"TETO开发"。

​**核心概念**：

- ​**事项 Item**：一个主题容器，有状态（活跃/推进中/放缓/停滞/已完成/已搁置）
- ​**文件夹 ItemFolder**：事项可以拖进文件夹分组（如"学习"、"开发"）
- ​**阶段 Phase**：事项下的时间阶段（如"基础期"、"冲刺期"），一个事项可以有多个阶段
- ​**目标 Goal**：跨事项的长期目标（如"今年读完12本书"）

**事项详情页** ﻿/items/[id]﻿：

- 左侧：事项基本信息 + 阶段时间线
- 右侧：该事项关联的记录区（自动按时间排列）

​**数据流向**：

```
plaintext记录 → 关联 item_id → 聚合到事项下
事项 → 拥有多个 Phase（阶段）
事项 → 可关联 Goal（目标）
事项 → 可放入 Folder（文件夹）
```

#### 3. 洞察页 ﻿/insights﻿ — 反馈闭环

​**用法**：看一段时间内的整体情况，回答"我这周/月做了什么？"。

​**四个统计区块**：

- ​**RecordStats**：记录总览（7天/30天数量、类型分布、标签分布、每日趋势）
- ​**ItemStats**：事项总览（活跃数、最活跃事项、停滞事项告警）
- ​**PhaseInsights**：阶段统计（最近阶段、状态分布）
- ​**GoalInsights**：目标统计（总目标数、状态分布）

​**时间范围选择**：7天 / 30天 / 本月 / 自定义

### 三、数据模型与关系

```
plaintext┌──────────────┐
│  record_days │ ← 按天容器（一天一个）
│  (date)      │
└──────┬───────┘
       │ 1:N
┌──────▼───────┐     N:1    ┌──────────┐
│   records    │───────────→│  items   │
│  (content,   │            │ (title,  │
│   type,      │            │  status) │
│   metric_*,  │            └────┬─────┘
│   duration)  │                 │ 1:N
└──────┬───────┘          ┌──────▼─────┐
       │ N:M              │  phases    │
┌──────▼───────┐          │ (阶段)     │
│     tags     │          └────────────┘
│  (标签)      │
└──────────────┘          ┌──────────┐
                          │  goals   │
  ┌──────────┐            │ (目标)   │
  │ folders  │←── items   └──────────┘
  │ (文件夹) │    放入
  └──────────┘
```

​**关键关系**：

- ﻿﻿record\_days﻿ → ﻿records﻿：一天有多条记录
- ﻿﻿records﻿ → ﻿items﻿：一条记录可选关联一个事项
- ﻿﻿records﻿ ↔ ﻿tags﻿：多对多标签
- ﻿﻿items﻿ → ﻿phases﻿：一个事项有多个阶段
- ﻿﻿items﻿ → ﻿goals﻿：事项可选关联目标
- ﻿﻿items﻿ → ﻿folders﻿：事项可选放入文件夹

### 四、技术栈与数据流

```
plaintext浏览器 (React/Next.js)
  │
  │  fetch /api/v2/...
  ▼
Next.js API Routes (src/app/api/v2/*)
  │
  │  调用 DB 层函数
  ▼
DB 层 (src/lib/db/teto-1.3/*.ts)
  │
  │  Supabase Client 查询
  ▼
Supabase PostgreSQL
  │
  │  RLS 校验 auth.uid() = user_id
  ▼
数据表
```

​**认证流程**：Supabase Auth → 登录后 JWT 自动附加到请求 → RLS 确保每个用户只能看到自己的数据。

### 五、典型使用流程

1. ​**日常记录**​：打开记录页 → 输入"跑步5公里" → 展开填统计(对象\=跑步, 数值\=5, 单位\=公里, 时长\=35分钟) → 提交
2. ​**关联事项**：编辑这条记录 → 选择关联到"健身计划"事项 → 保存
3. ​**事项追踪**：打开事项页 → 点进"健身计划" → 看到时间线上所有关联记录
4. ​**阶段推进**：在事项详情里新建阶段"第2月" → 设定起止日期
5. ​**回顾洞察**：打开洞察页 → 选"30天" → 看到跑步总公里数、活跃事项数、哪些事项停滞了
