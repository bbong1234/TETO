# 《第一轮发给 Trae 的 4 条正式提示词》

# 这一轮的目标只有一个：

> **把 TETO 1.0 的开发地基搭起来。**

不要急着做业务页面。  
先把：

- 项目初始化
- UI 基础
- Supabase 接入
- 登录认证

做好。

---

# 使用方式

建议你​**一条一条发给 Trae**，不要四条一起发。  
每做完一条，先本地运行确认，再发下一条。

并且每条最后都要求 Trae 输出：

1. 修改了哪些文件
2. 还需要我补哪些环境变量 / 配置
3. 如何本地验证
4. 下一步建议是什么

---

# 提示词 1：初始化项目

```
现在请你帮我初始化一个 TETO 1.0 项目。

项目背景：
TETO 是一个个人记录、日记复盘、项目跟踪和基础预测系统。当前只做 1.0 个人可用版。

本任务目标：
只完成项目初始化，不要扩展业务功能。

技术栈要求：
- Next.js（App Router）
- TypeScript
- Tailwind CSS

目录结构要求：
请使用 src 目录，并建立清晰结构，至少包含：
- src/app
- src/components
- src/lib
- src/types
- src/constants

任务要求：
1. 初始化一个可运行的 Next.js 项目
2. 开启 TypeScript
3. 配置 Tailwind CSS
4. 使用 src 目录结构
5. 保持代码结构清晰，避免加入无关示例业务代码
6. 首页先保留一个非常简单的占位页面即可

输出要求：
完成后请告诉我：
1. 你新增或修改了哪些文件
2. 当前项目目录结构
3. 如何本地启动
4. 下一步最适合做什么

注意事项：
- 不要加入数据库逻辑
- 不要加入认证逻辑
- 不要加入任何业务页面
- 不要自己扩展成复杂模板
- 先把项目地基搭好即可
```

# 提示词 2：集成 shadcn/ui 和基础布局

```
请在当前 TETO 1.0 项目中继续开发，完成 shadcn/ui 集成和基础后台布局搭建。

项目背景：
这是一个个人记录与项目管理系统，1.0 版本只做核心闭环，不扩展无关模块。

本任务目标：
只完成 UI 基础设施和静态导航，不接入业务数据。

技术栈要求：
- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui

任务要求：
1. 集成 shadcn/ui
2. 搭建全局基础 layout
3. 实现一个简洁的后台式页面结构
4. 提供主导航入口，至少包含以下页面入口：
   - Dashboard
   - Daily Record
   - Diary / Review
   - Projects
   - Stats
5. 以上页面先做静态路由占位页即可
6. 页面布局尽量简洁、清爽、数据产品风格，不要花哨
7. 尽量考虑后续扩展，但不要过度抽象

页面路径建议：
- /dashboard
- /daily-record
- /diary-review
- /projects
- /stats

输出要求：
完成后请告诉我：
1. 修改了哪些文件
2. 新增了哪些页面和组件
3. 当前导航结构
4. 如何本地验证布局是否正常
5. 后续接业务时哪些部分可以复用

注意事项：
- 先不要接 Supabase
- 先不要接登录
- 先不要写任何真实业务数据
- 不要加入复杂主题系统
- 不要加入无关动画和装饰
```

# 提示词 3：集成 Supabase

```
请在当前 TETO 1.0 项目基础上继续开发，完成 Supabase 接入准备。

项目背景：
TETO 1.0 将使用 Supabase 作为后端数据库和认证服务。本任务只做接入准备，不做具体业务表 CRUD。

本任务目标：
完成 Supabase 客户端集成、环境变量配置和基础工具封装。

技术要求：
- 使用 Supabase 官方 JS SDK
- 保持 Next.js App Router 兼容
- 环境变量不得写死在代码里

任务要求：
1. 安装 Supabase 相关依赖
2. 创建 Supabase 客户端初始化代码
3. 封装基础工具文件，放在清晰目录下（如 src/lib/supabase）
4. 提供 .env.example 示例文件
5. 告诉我需要配置哪些环境变量
6. 保持结构清晰，方便后续接数据库和认证
7. 可以加一个最简单的连接测试方式，但不要写业务查询逻辑

输出要求：
完成后请告诉我：
1. 修改了哪些文件
2. 新增了哪些 Supabase 工具文件
3. 需要我补充哪些环境变量
4. 如何验证 Supabase 是否接入成功
5. 下一步最建议做什么

注意事项：
- 不要创建业务表
- 不要写 daily_records 等业务代码
- 不要提前实现登录页面
- 不要把业务逻辑和 Supabase 初始化耦合在一起
```

# 提示词 4：实现登录认证

```
请在当前 TETO 1.0 项目基础上继续开发，完成基础登录认证能力。

项目背景：
TETO 1.0 是个人使用的记录与项目管理系统。当前需要先完成基础登录，保证后续用户数据隔离。

本任务目标：
基于 Supabase Auth 实现登录、登出、登录态检测和基础路由保护。

技术要求：
- Next.js App Router
- TypeScript
- Supabase Auth
- 页面风格保持简洁清爽

任务要求：
1. 实现登录页
2. 登录方式可使用邮箱登录或 Magic Link，优先简单稳定
3. 实现登出功能
4. 实现登录态检测
5. 未登录用户不能访问以下页面：
   - /dashboard
   - /daily-record
   - /diary-review
   - /projects
   - /stats
6. 登录成功后默认进入 /dashboard
7. 保持代码结构清晰，方便后续接业务数据
8. 页面不要复杂，但要可正常使用

输出要求：
完成后请告诉我：
1. 修改了哪些文件
2. 登录流程是如何实现的
3. 路由保护是如何实现的
4. 我需要在 Supabase 后台做哪些配置
5. 如何本地验证登录和登出流程

注意事项：
- 不要开始实现业务页面逻辑
- 不要加入用户资料编辑功能
- 不要加入复杂角色权限系统
- 不要扩展成多用户管理后台
- 只完成 TETO 1.0 当前需要的最小认证能力
```
