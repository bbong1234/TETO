# TETO 1.1 页面结构设计

## 一、文档定位

本文件用于定义 TETO 1.1 阶段的页面结构。

本文件负责：

- 明确 1.1 要做哪些页面
- 明确每个页面包含哪些主要区块
- 明确每个区块展示什么内容
- 明确每个页面主要读写哪些数据
- 明确页面之间的跳转关系
- 明确页面开发优先顺序

本文件不负责：

- 替代数据库设计文档
- 替代 SQL 文件
- 替代详细接口文档
- 替代组件级交互实现文档
- 替代视觉稿和 UI 细节稿

---

## 二、页面结构设计目标

TETO 1.1 页面结构需要支持以下能力：

1. DIY 任务定义
2. 任务目标设置
3. 每日任务记录
4. 任务完成度展示
5. 项目与任务关联
6. 一级子项目展示
7. Excel 导入
8. 页面区块拖拽排序
9. 单用户稳定使用
10. 后续可落地到真实 Supabase 数据结构

---

## 三、页面设计总原则

1. 先保证主闭环页面可用，再考虑扩展页面
2. 页面优先服务“任务 → 记录 → 目标 → 完成度 → 项目”
3. 第一版优先简单稳定，不追求页面过多
4. 每个页面只承担明确职责
5. 不把多个复杂工作流混在同一页
6. 优先保证数据可读、可录、可验证
7. 页面结构优先支持桌面端
8. 先支持固定区块，再考虑拖拽排序

---

## 四、1.1 页面总览

TETO 1.1 第一批核心页面建议为：

1. Dashboard（首页）
2. Tasks（任务页）
3. Task Detail / Task Edit（任务详情 / 编辑页）
4. Records（记录页）
5. Projects（项目页）
6. Project Detail（项目详情页）
7. Import（导入页）
8. Layout Settings（布局偏好页，可后做）

说明：

- 若第一版想更稳，可以先不单独做 Layout Settings 页面
- 布局排序也可以先直接挂在 Dashboard 页面内完成
- 但页面结构设计中仍保留布局偏好能力

## 5.1 Dashboard（首页）

### 页面定位

首页用于提供 1.1 的总览入口，是用户进入系统后的第一主页面。

它负责：

- 看今日任务情况
- 看今日完成度
- 看项目整体情况
- 快速进入记录
- 快速进入任务和项目管理

### 页面区块建议

#### 1）顶部概览区

展示内容：

- 今日日期
- 今日任务总数
- 今日已记录任务数
- 今日完成度总览
- 快捷操作按钮

快捷操作按钮建议：

- 新建任务
- 记录今日任务
- 进入项目页
- 进入导入页

#### 2）今日任务概览区

展示内容：

- 今日需要关注的任务列表
- 每个任务当前记录值
- 每个任务目标值
- 每个任务完成度
- 快捷录入按钮

主要数据来源：

- ​`task_definitions`
- ​`task_daily_records`
- ​`task_targets`

#### 3）今日完成度区

展示内容：

- 今日整体完成度
- 参与计算的任务数
- 无目标任务数
- 已完成 / 未完成任务数

说明：

- 若第一版不做复杂图表，可先使用数字卡片 + 简单列表

#### 4）项目完成度概览区

展示内容：

- 项目列表
- 每个项目完成度
- 子项目数量
- 项目下任务数量

主要数据来源：

- ​`project_nodes`
- ​`task_definitions`
- ​`task_targets`
- ​`task_daily_records`

#### 5）近期记录区

展示内容：

- 最近 7 天或最近若干条任务记录摘要
- 最近更新任务
- 最近补录记录

主要数据来源：

- ​`task_daily_records`

#### 6）页面布局区块排序

说明：

- Dashboard 支持区块顺序拖拽
- 第一版只允许固定区块排序
- 不做自由网格布局

主要数据来源：

- ​`user_layout_preferences`

### 首页主要作用

首页不承担复杂配置，只承担：

- 看总览
- 做快速进入
- 做快捷录入

---

## 5.2 Tasks（任务页）

### 页面定位

任务页用于管理所有任务定义。

它负责：

- 查看任务列表
- 新建任务
- 编辑任务
- 停用任务
- 设置任务归属项目
- 查看任务目标状态

### 页面区块建议

#### 1）任务页顶部操作区

展示内容：

- 新建任务按钮
- 搜索框
- 状态筛选
- 项目筛选
- 任务类型筛选

#### 2）任务列表区

每条任务建议展示：

- 任务名称
- 任务类型
- 单位
- 关联项目
- 是否参与完成度
- 当前状态
- 当前目标摘要
- 编辑按钮
- 停用按钮

主要数据来源：

- ​`task_definitions`
- ​`task_targets`
- ​`project_nodes`

#### 3）任务快速统计区

展示内容：

- 总任务数
- 启用任务数
- 停用任务数
- 有目标任务数
- 无目标任务数

#### 4）批量操作区（可后做）

第一版可先不做复杂批量操作。  
如果要做，最多支持：

- 批量停用
- 批量挂项目

### 任务页主要作用

任务页核心是“管理定义”，不是“记录每日值”。  
每日值录入应主要在记录页或快捷录入中完成。

---

## 5.3 Task Detail / Task Edit（任务详情 / 编辑页）

### 页面定位

该页面用于查看和编辑某一个任务的完整配置。

### 建议包含的模块

#### 1）基础信息模块

字段建议：

- 任务名称
- 描述
- 任务类型
- 单位
- 展示单位
- 存储单位

主要数据来源：

- ​`task_definitions`

#### 2）记录规则模块

字段建议：

- 记录模式
- 是否允许快捷输入
- 是否允许手动输入
- 是否允许补录

#### 3）统计规则模块

字段建议：

- 是否参与统计
- 是否参与完成度计算

#### 4）项目归属模块

字段建议：

- 所属项目
- 所属子项目（如有）

主要数据来源：

- ​`project_nodes`

#### 5）目标设置模块

展示与编辑：

- 日目标
- 周目标
- 月目标
- 年目标
- 自定义目标

主要数据来源：

- ​`task_targets`

#### 6）近期记录摘要模块

展示：

- 最近 7 天该任务的记录
- 最近完成度变化

主要数据来源：

- ​`task_daily_records`

### 页面作用

- 作为任务配置中心
- 允许集中修改任务与目标
- 允许查看该任务近期记录情况

---

## 5.4 Records（记录页）

### 页面定位

记录页用于按日期查看和编辑任务记录。

它是 1.1 任务系统的核心操作页之一。

### 页面区块建议

#### 1）日期切换区

展示内容：

- 当前日期
- 上一天 / 下一天
- 日期选择器
- 回到今天

#### 2）当日任务记录列表区

每行展示：

- 任务名称
- 任务单位
- 目标值
- 当前已记录值
- 完成度
- 备注入口
- 保存按钮或行内编辑入口

主要数据来源：

- ​`task_definitions`
- ​`task_daily_records`
- ​`task_targets`

#### 3）快捷录入区

适合首页或记录页顶部快捷操作：

- boolean 任务直接勾选
- count / number 直接输入数值
- duration 输入分钟或小时
- amount 输入金额

#### 4）记录状态摘要区

展示内容：

- 今日已记录任务数
- 未记录任务数
- 无目标任务数
- 今日总完成度

#### 5）补录入口

说明：

- 用户可切换历史日期录入
- 第一版支持补录，但不做复杂审核流

### 页面作用

该页面负责：

- 按日查看任务
- 按日填写或修改任务值
- 做最核心的每日输入闭环

---

## 5.5 Projects（项目页）

### 页面定位

项目页用于管理 1.1 新项目体系。

它负责：

- 展示主项目
- 展示一级子项目
- 展示项目下任务数量
- 查看项目完成度
- 新建 / 编辑 / 归档项目

### 页面区块建议

#### 1）顶部操作区

展示内容：

- 新建项目按钮
- 搜索框
- 状态筛选
- 主项目 / 子项目筛选

#### 2）项目列表区

每个项目建议展示：

- 项目名称
- 层级标识（主项目 / 子项目）
- 父项目名称
- 任务数量
- 参与统计任务数
- 当前完成度
- 状态
- 编辑按钮

主要数据来源：

- ​`project_nodes`
- ​`task_definitions`
- ​`task_targets`
- ​`task_daily_records`

#### 3）项目摘要区

展示内容：

- 主项目数量
- 子项目数量
- 有任务项目数量
- 无任务项目数量

### 页面作用

项目页以“管理项目结构”为主，  
不是旧版那种“项目进度手工日志页”。

---

### 页面模块建议

展示：

- 项目名称
- 描述
- 父项目信息
- 状态

主要数据来源：

- ​`project_nodes`

#### 2）子项目列表模块

展示：

- 该主项目下的一级子项目

#### 3）项目任务列表模块

展示：

- 挂在该项目下的任务
- 每个任务目标状态
- 每个任务当前完成度
- 快捷跳转到任务详情

主要数据来源：

- ​`task_definitions`
- ​`task_targets`
- ​`task_daily_records`

#### 4）项目完成度模块

展示：

- 项目整体完成度
- 参与统计任务数
- 无目标任务数
- 最近完成度变化

### 页面作用

- 查看单个项目
- 查看项目下任务
- 看项目维度完成度

---

## 5.7 Import（导入页）

### 页面定位

导入页用于支持 Excel / 表格批量导入任务记录。

### 页面流程建议

#### 第一步：上传文件

展示：

- 文件上传区域
- 模板下载入口
- 导入说明

#### 第二步：预览映射结果

展示：

- 识别到的列
- 映射结果
- 不识别字段提示
- 错误行提示

#### 第三步：冲突检查

展示：

- 命中已有记录的行
- 冲突类型
- 是否继续导入

规则：

- 第一版不自动覆盖
- 冲突先提示

#### 第四步：导入结果

展示：

- 成功条数
- 失败条数
- 错误摘要
- 可选的失败详情

主要数据来源：

- ​`import_batches`
- ​`task_definitions`
- ​`task_daily_records`

### 页面作用

- 提供固定模板导入入口
- 负责导入预览和导入结果反馈
- 第一版不做万能导入器

## 七、页面跳转关系设计

### 核心跳转关系

1. Dashboard → Tasks
2. Dashboard → Records
3. Dashboard → Projects
4. Dashboard → Import
5. Tasks → Task Detail / Edit
6. Tasks → Records（按任务跳转可选）
7. Projects → Project Detail
8. Project Detail → Task Detail / Edit
9. Import → 导入结果页（可复用同页流程，不一定拆独立页）

### 设计原则

- 首页负责分发
- 列表页负责筛选与进入详情
- 详情页负责集中编辑
- 记录页负责日常高频输入

---

## 八、页面优先开发顺序

### 第一优先级

1. Records
2. Tasks
3. Task Detail / Edit

原因：

- 先跑通任务定义与每日记录主闭环

### 第二优先级

4. Dashboard
5. Projects
6. Project Detail

原因：

- 在主闭环跑通后，再做总览与项目汇总

### 第三优先级

7. Import
8. Layout Settings

原因：

- 导入和布局属于增强能力，不是主闭环最先阻塞项

---

## 九、页面开发最小闭环

TETO 1.1 第一批最小可用页面闭环建议为：

1. Tasks
2. Task Detail / Edit
3. Records

只要这三块跑通，就已经可以完成：

- 建任务
- 设目标
- 记每日值
- 看任务完成情况

这是 1.1 最核心的可验证闭环。

## 十、当前明确不在本轮展开的页面

本轮先不展开以下页面或复杂能力：

1. 多人协作页面
2. 权限管理页面
3. 无限层级项目树页面
4. 自动化规则配置页面
5. AI 智能解析配置页面
6. 自定义可视化报表中心
7. 移动端专用页面结构
8. 复杂拖拽网格编辑器
9. 组件级低代码页面搭建器

---

## 十一、页面结构拍板结论

TETO 1.1 页面结构第一版应围绕以下闭环展开：

> 任务页 → 任务详情页 → 记录页 → 首页总览 → 项目页 → 导入页

其中最核心的落地顺序是：

1. 先做任务定义
2. 再做每日记录
3. 再做任务目标联动
4. 再做首页汇总
5. 再做项目汇总
6. 最后补导入和布局偏好

---

## 十二、最终定义

TETO 1.1 页面结构当前正式定义为：

> **以任务定义、每日记录、目标值、完成度、项目归属为核心，优先构建 Tasks / Task Detail / Records 三页最小闭环，在此基础上逐步补齐 Dashboard、Projects、Import 和布局偏好能力。**
