### 一、事项的定义

> **事项是一个长期主题容器。它代表你人生中一个持续存在的主题领域。**

事项回答的问题是：**我长期在处理什么主题？**

**事项的例子：**

- 英语
- 健身
- 工作
- TETO 开发
- 抖音运营

**事项不是：**

- 一个具体行动（“背50个单词”是记录）
- 一个期望结果（“通过四级”是目标）
- 一个时间段概括（“四级备考期”是阶段）

---

### 二、事项下面挂什么

一个事项下面有四种对象，分属四个维度：

| 对象 | 维度     | 回答的问题           | 例子                   |
| ------ | ---------- | ---------------------- | ------------------------ |
| **子项**     | 行动维度 | 我具体在做什么？     | 背单词、练听读、练口语 |
| **目标**     | 方向维度 | 要做到什么程度？     | 每天110个、通过四级    |
| **阶段**     | 时间维度 | 这段时期整体怎么样？ | 四级备考期、基础积累期 |
| **记录**     | 事实维度 | 某天具体发生了什么？ | 今天背了50个单词       |

**四者关系图：**

text

```
事项：英语（主题容器）
│
├── 子项 1：背单词（行动线）
│   ├── 目标：量化型 — 每天110个
│   ├── 目标：重复型 — 每天完成
│   └── 记录（所有 metric_name='单词' 的记录）
│
├── 子项 2：练听读（行动线）
│   ├── 目标：量化型 — 每天30分钟
│   └── 记录（所有 metric_name='英语听读' 的记录）
│
├── 子项 3：练口语（行动线）
│   ├── 目标：重复型 — 每周3次
│   └── 记录（所有 sub_item_id='练口语' 的记录）
│
├── 目标：达标型 — 通过四级（挂在事项上的大目标）
│
├── 阶段 1：基础积累期（2024/12 — 2025/6）
├── 阶段 2：四级备考冲刺期（2025/7 — 2025/12）
│
└── 所有记录（不管属于哪个子项，都挂在事项下）
```

### 三、子项

#### 3.1 子项的定义

> **子项是事项下面的一条持续的行动线。它回答的是：“在这个主题下，我具体在做什么？”**

#### 3.2 子项与目标的核心区别

| 对比       | 子项                 | 目标                         |
| ------------ | ---------------------- | ------------------------------ |
| 回答的问题 | 我在做什么？         | 要做到什么程度？             |
| 例子       | 背单词、练听读       | 每天110个、通过四级          |
| 性质       | 行动线（持续存在）   | 期望结果（可达成、可变更）   |
| 生命周期   | 长期存在，直到不再做 | 达成后归档，新目标替换旧目标 |

#### 3.3 子项的创建原则

- **可选的**：当事项只有一个行动线时（如“冥想”事项只有“每日冥想”），不需要建子项，事项即子项
- **多个行动线**：当事项下出现两条以上不同性质的持续行动线时，就建子项
- **子项不能跨事项存在**：一个子项只能属于一个事项
- **子项的名称可随时间演变**：支持改名，但历史记录的 `sub_item_id` 不变

#### 3.4 子项不需要独立阶段

阶段统一挂在事项上，所有子项共享。因为阶段是对“整个事项”在某段时期的概括。

子项自身的状态变化（如某子项在某个阶段是停滞的），通过子项的数据看板体现，不单独建阶段。

#### 3.5 子项升级为独立事项

**触发条件：**  子项有了独立的生命周期（自己的起止时间、节奏），需要被单独拿出来查看自己的时间线。

**操作流程：**

1. 用户手动触发“升格为独立事项”
2. 系统基于子项信息创建新事项（标题、描述继承）
3. 历史记录默认迁移到新事项（用户可选不迁移）
4. 原子项保留在原事项下（它是原事项历史的一部分，不能被删除）

### 四、目标

#### 4.1 目标的三种类型

| 类型 | 定义                                       | 关键字段                                                 | 例子                  |
| ------ | -------------------------------------------- | ---------------------------------------------------------- | ----------------------- |
| **量化型**     | 有数值指标，有期望值，有起算日期           | 关联指标名、单位、每日期望值、起算日期、截止日期（可选） | 每天背110个单词       |
| **达标型**     | 是/否完成，有明确边界                      | 状态（进行中/已达成/已放弃/已暂停）                      | 通过四级              |
| **重复型**     | 每天/每周/每月完成指定次数，只记录完成与否 | 重复频率（每天/每周/每月）、次数                         | 每天冥想、每周跑步3次 |

#### 4.2 目标归属规则

- **量化型目标**：挂在子项下
- **重复型目标**：挂在子项下
- **达标型目标**：可以挂在子项下（如“完成听力教程”），也可以挂在事项上（如“通过四级”）

#### 4.3 目标变更规则（防数据回溯）

**核心原则：历史数据绝不变动。**

| 场景                                       | 处理方式                                                                             |
| -------------------------------------------- | -------------------------------------------------------------------------------------- |
| 修改写错的目标（110写成了1100）            | 直接修改，历史数据重新对比修正后的标尺                                               |
| 目标升级（1万粉 → 10万粉）                | 旧目标标记为“已达成”，系统提示创建新阶段，在新阶段下创建新目标。新目标从零开始计算 |
| 目标替换（从“通过四级”换成“通过六级”） | 旧目标标记为“已达成”，创建新目标                                                   |
| 放弃目标                                   | 旧目标标记为“已放弃”，保留历史数据不删除                                           |

**量化型目标升级操作流程：**

1. 将旧目标状态改为“已达成”，记录达成日期
2. 系统提示：“要以此为新起点，创建一个新阶段吗？”
3. 用户确认 → 创建新阶段
4. 在新阶段下创建新目标
5. 新目标从零开始计算差额，旧目标数据永远定格

### 五、阶段

#### 5.1 阶段的定义

> **阶段是人对事项某段时期的主观概括。它回答的是：“这件事在这一段时期整体是什么样子？”**

#### 5.2 阶段和子项的关系

- 阶段是时间维度，子项是行动维度。两者正交，互不替代
- 一个事项的阶段是所有子项共享的时间背景
- 子项不需要独立的阶段

**阶段和子项在时间轴上的关系：**

text

```
时间轴（从左到右）
├── [阶段：基础积累期] ──────────────────┤├── [阶段：四级备考期] ────────────┤
│  背单词：每天50个                       │  背单词：每天110个                  │
│  练听读：每天20分钟                     │  练听读：每天30分钟                  │
│  练口语：每周2次                        │  练口语：停滞                        │
```

#### 5.3 阶段的来源

| 来源 | 说明                                     | 触发条件                   |
| ------ | ------------------------------------------ | ---------------------------- |
| **手动创建**     | 用户主动命名一段时期                     | 任何时间                   |
| **目标达成时引导**     | 系统提示“要以此为新起点创建新阶段吗？” | 目标状态变为“已达成”     |
| **从记录生成**     | 系统分析记录密度变化，给出客观建议       | 用户点击“从记录生成”按钮 |

**从记录生成的规则：**

- 检测记录密度变化点，生成客观建议（如“近30天记录密度较前30天显著增加”）
- 禁止生成带有主观评价的标题（“懈怠期”、“低效期”）
- 生成的标题必须是客观描述（“高活跃期”、“低密度期”、“恢复期”）
- 用户确认或修改后才创建阶段
- 与已有阶段时间范围重叠超过70%时不返回
- 一次最多返回3条建议

#### 5.4 阶段在时间轴上的展示

- 阶段在时间轴上以独立的标记行展示，与记录行交错排列
- 记录通过日期匹配自动显示所属阶段的标签
- 即使某个阶段内没有记录，阶段标记行仍然显示（因为补充阶段的意义就是“这段时期存在但无记录”）

### 六、核心原则

**原则一：唯一归属原则**
一条记录、一个子项，在任一时刻，只能属于唯一一个事项。如果发现归属模糊，说明事项结构需要调整，而不是去建立复杂的多对多关系。

**原则二：独立判断原则**
一个子项是否应该独立成事项，不看它的大小，只看它是否有独立的生命周期（自己的起止时间、节奏、阶段变化）。

**原则三：文件夹收纳原则**
“英语”和“编程”这种平级但同属“学习”大类的事项，用文件夹收纳它们的关系。文件夹只是分组视图，不承载阶段、目标、记录。

**原则四：主题本质原则**
一件事如果因为环境、身份、目标的重大变化而改变了本质（如同一项目从A公司转到B公司），应视为两个独立事项。原来事项归档，新事项从零开始。

### 七、事项页界面结构

text

```
事项详情页
├── Header（事项名称、状态、描述）
├── 数据总览
│   ├── 基础趋势（近7天/30天记录数、时长、趋势图）
│   └── 目标对照（各子项下的目标仪表盘，仅当有目标时显示）

│   ├── Tab：全部 │ 背单词 │ 练听读 │ 练口语
│   ├── 切换 Tab 后，下方数据看板和记录列表切换到对应子项
│   └── 新建子项入口
├── 目标管理区
│   ├── 量化型/重复型目标列表（按子项分组）
│   ├── 达标型目标列表（事项级大目标）
│   └── 新建目标入口
├── 阶段管理区
│   ├── 当前阶段醒目展示
│   ├── 阶段列表 + 新建/从记录生成入口

    ├── 阶段标记行 + 记录行交错展示
    ├── 记录按时间降序排列
    └── 记录通过日期匹配显示所属阶段标签
```

### 八、事项生命周期

| 操作 | 行为                                                             |
| ------ | ------------------------------------------------------------------ |
| **创建事项**     | 检查是否有同名归档事项，有则提示“是否在原事项下建新阶段重启？” |
| **完成事项**     | 状态改为“已完成/已归档”，所有子项、阶段、目标、记录保留        |
| **删除事项**     | 关联记录的 `item_id` 置空（不级联删除记录），禁止物理删除                 |
| **重启事项**     | 在原事项下建新阶段，不新建同名事项                               |

### 九、事项建模速查表

| 场景                                 | 建模方式                          | 理由                         |
| -------------------------------------- | ----------------------------------- | ------------------------------ |
| 英语（背单词+练听读+练口语）         | 一个事项 + 3个子项                | 共享“英语”主题和阶段       |
| 冥想（只有每日冥想）                 | 一个事项，无子项                  | 单一行动线，事项即子项       |
| 爬山、滑雪（同属户外运动但各自独立） | 两个独立事项 + “户外运动”文件夹 | 平级主题，文件夹收纳         |
| A公司XX项目 → B公司继续该项目       | 两个独立事项                      | 身份/目标/环境变了，本质不同 |
| 工作（同时跟进多个项目）             | 一个事项 + 多个子项               | 子项可在时间轴上对齐         |
| 某个子项有独立生命周期               | 升级为独立事项                    | 有独立阶段 = 值得独立        |
| 英语课时长+课节数                    | 一个子项 + 拆成两条记录           | 同一行动线的两个指标         |

### 十、三项关键规则

**规则一：子项优先级**
当你不确定该建子项还是独立事项时，优先建子项。因为子项可以随时升级为事项，但事项降级为子项需要数据迁移。

**规则二：目标绑定**
量化型目标和重复型目标必须挂在子项下。达标型目标可灵活选择。创建目标时，系统自动识别当前选中的子项并绑定。

**规则三：数据不可逆**
目标一旦标记为“已达成”，其数据永久定格，不可修改。子项升级为事项后，原事项下的原子项保留，历史记录不强制迁移。
