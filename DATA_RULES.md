# 数据规则文档

## 1. 数据真源定义

### 1.1 任务管理 = 配置真源
负责存：
- 任务名
- 类型
- 单位
- 是否参与统计
- 起始日期
- 日目标 / 周目标 / 月目标
- 项目归属
- 状态
- 排序

### 1.2 今日记录 = 原始事实真源
负责存：
- 日期
- task_id
- 完成状态
- 数值
- 备注
- 情绪 / 状态（如有）

### 1.3 记录总表 = 原始记录总览视图
- 不是第二真源，不是独立统计源，只是查看、汇总、筛选入口

### 1.4 统计分析 = 聚合计算结果
- 没有独立结果真源表，必须基于任务配置和原始记录统一计算得出

## 2. 页面职责

### 2.1 今日记录页
- 只负责记录当天内容：今日任务录入、今日总结、今日状态
- 今日完成度只能按当天值算

### 2.2 记录总表
- 只负责展示全部原始记录，不负责成为统计真源

### 2.3 任务管理页
- 只负责配置任务规则，不负责存真实完成值

### 2.4 统计分析页
- 只负责做周/月/累计聚合分析，不单独维护第二套数据

## 3. 开始时间规则

优先级：
1. 手动配置的任务起始日期
2. 首次有效记录日期（手动录入的0也算有效记录）
3. 任务创建时间

## 4. 目标值规则

当前支持：
- 日目标
- 周目标
- 月目标

优先级：
- 有直接配置优先用直接配置
- 没有再推导

具体：
- 本周目标：有周目标用周目标，否则日目标 × 7
- 本月目标：有月目标用月目标，否则日目标 × 当月天数

## 5. 完成值规则

- 今日完成：当天记录汇总
- 本周完成：本周记录汇总
- 本月完成：本月记录汇总
- 累计完成：从开始时间到当前统计日期的记录汇总

## 6. 统计规则

### 6.1 差额和进度
普通正向任务：
- 差额 = 目标 - 完成
- 进度 = 完成 / 目标 × 100%
- 允许超过 100%

### 6.2 任务类型口径

当前真实已有类型：
- 完成/未完成
- 次数型
- 数值型

补充规则：

1. 完成/未完成型
   - 已完成 = 1
   - 未完成 = 0

2. 次数型
   - 按次数累加

3. 数值型
   - 直接累加数值

4. 备注型
   - 只记录
   - 不参与统计

5. 情绪类
   - 后续如果进入统计，必须先数值化，不要直接乱入统计逻辑

## 7. 当前支持任务类型

- 完成/未完成型（boolean）
- 次数型（count）
- 数值型（number）

## 8. 当前不支持内容

- 时间点型任务（如起床时间、睡觉时间）
- 上限型任务（如游戏不超过60分钟）
- 情绪统计复杂扩展
- Excel 导入架构级重构
- 本地为真源
- 双端同步冲突系统
- 更复杂的阶段目标系统
- 未来预埋功能

## 9. 后续功能方向

### 9.1 任务类型扩展

1. 时间点型任务
   - 支持“起床时间”“睡觉时间”这类任务
   - 记录时间点，不适合直接做简单求和

2. 上限型任务
   - 支持“不要超过 xx 分钟 / xx 次”这类任务
   - 逻辑与普通正向任务相反

3. 情绪数值化任务
   - 让情绪也能进入统计
   - 必须先做数值映射

### 9.2 统计分析能力增强

- 趋势图
- 项目维度汇总
- 任务详情抽屉更完整信息
- 达标率统计
- 周/月对比
- 最近连续完成天数

### 9.3 导入能力增强

- 本地解析再上传
- 导入前预览
- 类型校验
- 错误行提示
- 分批上传
- 导入结果摘要
- 导入后自动刷新相关页面数据

### 9.4 本地体验增强

- 本地缓存增强
- 本地暂存
- 待同步队列（未来）

### 9.5 多端同步（未来）

- 冲突处理
- 更新时间判断
- 覆盖策略
- 合并策略
